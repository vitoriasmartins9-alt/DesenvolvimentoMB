Presentation


Qual Objetivo Principal? O objetivo é entregar uma experiência de usuário (UX) rica, otimizada para as características únicas dos dispositivos móveis, incluindo telas menores e sensíveis ao toque. Conectividade (Wi-Fi, 4G/5G). Recursos de hardware nativos (câmera, GPS, acelerômetro, etc.). Limitações de bateria e processamento.

Nativo, O aplicativo é construído usando linguagens e ferramentas específicas para cada sistema operacional. Oferece o melhor desempenho e acesso total aos recursos do dispositivo. iOS: Swift, Objective-C. Android: Kotlin, Java.

Cross-Platform, Usa um único código-base que é compilado para rodar em ambas as plataformas (iOS e Android), geralmente traduzindo o código em elementos nativos da UI. Flutter (Dart), React Native (JavaScript/TypeScript).

Híbrido, Usa tecnologias web (HTML, CSS, JavaScript) dentro de um "contêiner" nativo (Web View). É rápido de desenvolver, mas pode ter um desempenho inferior ao nativo. Ionic, Cordova.

Steps

Etapa 1-Desenvolvimento do Front-end: É o código que controla a interface do usuário (UI) e a interação dentro do dispositivo. É o que o usuário vê e toca.

Etapa 2-Desenvolvimento do Back-end (Opcional): Muitas aplicações precisam de um servidor para armazenar dados, gerenciar autenticação de usuários e executar a lógica de negócios central. Isso é feito por meio de APIs.

Etapa3-Publicação e Manutenção: Inclui a submissão do aplicativo às lojas de aplicativos e a contínua atualização e correção de bugs (manutenção) após o lançamento.
